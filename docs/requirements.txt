============
Requirements
============

Draft, to be implemented in `0.8.4`.

1. Input and output on standard input, uses `html` writer (default)::

       $ cat infile.rst | flexirest

2. Input from file `infile.rst`, output on standard output, `html` writer::

       $ flexirest infile.rst

3. Input from file `infile.rst`, output in file `outfile.html`. `html`
   writer::

       $ flexirest html infile.rst outfile.html

4. Input from file `infile.rst`, output in file `outfile.pdf`,
   `pdflatex` writer. Use the stylesheet mystyle that is either the
   file mystyle.sty in the current directory or findable by your TeX
   version (e.g. in the appropriate subdirectory of your
   `$TEXMFLOCAL`)::

       $ flexirest pdflatex --stylesheet=mystyle infile.rst outfile.pdf

5. Run hypotetical `reStructuredText` generating command `rest-gen`,
pipe it's output to flexirest and write result in `out.pdf` using the
`RTF` writer::

       $ flexirest rtf -o out.rtf < `rest-gen`

6. Print current version and exit (same behaviour as `0.8.2` and earlier)::

       $ flexirest --version

7. Print help summary and exit (same behaviour as `0.8.2` and
   earlier)::

       $ flexirest --help

8. Dumps all parts generated by `docutils` for writer `latex` when
   running on `infile.rst`, input from standard input::

       $ flexirest latex --dump-parts

9. Input from `infile.rst`, dumps only the `'body'` part generated by
   `docutils` for the `latex` writer, input from file `infile.rst`::

       $ flexirest latex --dump-part=body infile.rst
